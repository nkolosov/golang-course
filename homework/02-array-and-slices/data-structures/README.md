# Структуры данных

## Описание задачи

Нужно реализовать пакет и тесты для работы со структурами данных
### Двусвязный список
Для двусвязного списка необходимо поддержать следующие методы:

- `PushFront` добавляет элемент в начало списка
- `PopFront` удаляет и возвращает первый элемент
- `PushBack` добавляет элемент в конец списка
- `PopBack` удаляет и возвращает последний элемент
- `InsertAfter` добавляет элемент после указанного узла
- `InsertBefore` добавляет элемент перед указанным узлом
- `AddAtIndex` добавляет элемент по указанному индексу
- `RemoveAtIndex` удаляет элемент по указанному индексу
- `Clear` удаляет все элементы
- `GetIndex` возвращает индекс указанного элемента
- `All` возвращает все элементы при помощи 
- `Range` возвращает функцию-итератор `iter.Seq`

### Стек
Для стека необходимо поддержать следующие методы:

- `Push` добавляет элемент на вершину стека
- `Pop` удаляет и возвращает элемент с вершины стека
- `Peek` возвращает верхний элемент без его удаления
- `Size` возвращает количество элементов в стеке 
- `Clear` очистить стек

## Очередь
Для очереди необходимо поддержать следующие методы:

- `Enqueue` добавляет элемент в конец очереди
- `Dequeue` удаляет и возвращает элемент в начале очередь
- `Peek` возвращает первый элемент в очереди элемент без его удаления
- `Size` возвращает количество элементов в очереди
- `Clear` очистить очередь

Весь функционал коллекций должен быть покрыт тестами

### Запуск проекта
Работающий `main.go` не требуется

### Детали реализации
- Проект должен соответствовать Go Project Layout
- В проекте должен быть .gitignore, Makefile, go.mod
- К проекту должен быть подключен линтер https://golangci-lint.run/ и в проекте должен находиться файл конфигурации .golangci-lint.yaml со стандартным набором линтероа
```yaml
version: "2"

linters:
  default: standard
```
- Для тестирования необходимо использовать пакет [testify](https://github.com/stretchr/testify)
- Тесты необходимо сделать в формате табличных тестов, где это возможно
